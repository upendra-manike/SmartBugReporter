Privacy Policy – Smart Bug Reporter

Last updated: March 2025

This privacy policy applies to the Smart Bug Reporter Chrome extension (“the extension”).

1. What the extension does

When you click “Capture bug” or use the auto-test flow, the extension captures a screenshot of the current tab, console logs and network requests from that page, and browser/environment information. It then formats this into a report that you can copy and paste into Jira or GitHub. The extension is intended for developers, QA, and product teams.

2. What data is used

Only when you actively start a capture, the extension may:

  • Capture a screenshot of the visible area of the current tab.
  • Read console logs (log, info, warn, error, debug) that our script has stored from the page.
  • Read network request data (URL, method, status, duration) that our script has stored from the page.
  • Read browser/environment info (page URL, User-Agent, platform, screen and viewport size).

Any title and steps to reproduce you type in the popup are included only in the report you copy. They are not sent to us or stored on our servers.

3. Where data is stored

Capture data is passed to the extension popup so you can copy the report. It is not stored permanently on our servers or synced anywhere. The extension does not use Chrome’s storage API; nothing is persisted beyond your current session in the browser.

4. Data sharing and transfer

We do not:

  • Sell, rent, or share your data with third parties.
  • Send screenshots, logs, or network data to any remote server.
  • Use your data for advertising, analytics, or credit/lending purposes.

You choose where to paste the report (e.g. your own Jira or GitHub).

5. Your control

You decide when to start recording or click “Capture bug.” No capture or data collection happens in the background. You can close the popup without copying if you do not want to use the report.

6. Permissions

The extension requests:

  • activeTab – To take a screenshot of the current tab and read capture data when you start a capture.
  • Host permissions – So our content script can run on the page you are viewing to capture console and network data, and so we can run a script when you capture. We do not access sites in the background.
  • scripting – To run a function in the current tab when you capture to collect the stored console and network data.

7. Changes

We may update this policy from time to time. The “Last updated” date at the top will be revised when we do. Continued use of the extension after changes means you accept the updated policy.

8. Contact

For questions about this privacy policy or the extension, use the contact email provided in the Chrome Web Store listing for Smart Bug Reporter.
